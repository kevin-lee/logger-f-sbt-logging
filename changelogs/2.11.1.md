## Bug Fixed

### The released jar contains no binary (#5)

Restructure the `logger-f-sbt-logging` module source layout so the published jar actually contains the compiled binary.

The previous `shared/` cross-build layout resulted in an empty jar, so the sources are moved into the standard module layout and split by Scala version:

- Extract the version-agnostic logger into `SbtLoggerBase` and provide Scala 2 / Scala 3 specific `SbtLogger` companions (`implicit` vs `using`).
- Rename the root project to `loggerFSbtLoggingRoot` (`sbt-logging-root`) to distinguish it from the actual library module.

Plugin changes:

- Add `sbt-tpolecat` `0.5.7`
- `sbt-devoops`: `3.3.2` => `3.5.1`
